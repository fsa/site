# Внедрение IPv6 в сетях провайдеров: руководство с опорой на стандарты и рекомендации

## Введение

Переход на IPv6 уже давно перестал быть вопросом «надо или нет» и стал задачей «как и когда». Для операторов связи и интернет-провайдеров внедрение IPv6 — это не только вопрос исчерпания IPv4-адресов, но и обеспечение долгосрочной масштабируемости, упрощение маршрутизации и повышение качества услуг.

В этой статье я собрал подборку ключевых **RFC**, **BCP** и **BCOP**, которые пригодятся инженерам при проектировании и внедрении IPv6 в сетях доступа и магистрали.

---

## 1. Базовые стандарты IPv6

Перед проектированием IPv6-сети важно понимать, на каких базовых стандартах она построена:

| Документ     | Название (EN)                                       | Перевод (RU)                          | Назначение                                                              |
| ------------ | --------------------------------------------------- | ------------------------------------- | ----------------------------------------------------------------------- |
| **RFC 8200** | *Internet Protocol, Version 6 (IPv6) Specification* | Спецификация интернет-протокола IPv6  | Описывает структуру IPv6-пакета, заголовки, обработку маршрутизаторами. |
| **RFC 4291** | *IPv6 Addressing Architecture*                      | Архитектура адресации IPv6            | Определяет типы IPv6-адресов, форматы и правила их использования.       |
| **RFC 4861** | *Neighbor Discovery for IPv6 (ND)*                  | Обнаружение соседей в IPv6            | Механизм определения соседей, маршрутизаторов и доступных префиксов.    |
| **RFC 4862** | *IPv6 Stateless Address Autoconfiguration (SLAAC)*  | Автоконфигурация адреса без состояния | Механизм автоматической настройки адреса узла без DHCP.                 |
| **RFC 8201** | *Path MTU Discovery for IPv6*                       | Определение MTU пути в IPv6           | Рекомендации по настройке MTU для избежания фрагментации.               |

## 2. Рекомендации по внедрению IPv6 (BCP)

Best Current Practice (BCP) — это не формальные стандарты, а набор проверенных рекомендаций, которыми руководствуются операторы и инженеры.

| Документ               | Название (EN)                                           | Перевод (RU)                                 | Применение                                            |
| ---------------------- | ------------------------------------------------------- | -------------------------------------------- | ----------------------------------------------------- |
| **BCP 38 / RFC 2827**  | *Network Ingress Filtering*                             | Фильтрация входящего трафика                 | Предотвращение IP-спуфинга и отражённых атак.         |
| **BCP 84 / RFC 3704**  | *Ingress Filtering for Multihomed Networks*             | Фильтрация в мультихом-сетях                 | Защита при использовании нескольких провайдеров.      |
| **BCP 194 / RFC 7157** | *IPv6 Multihoming without Network Address Translation*  | IPv6-мультихоминг без NAT                    | Рекомендации по мультихом-сетям IPv6.                 |
| **BCP 198 / RFC 7381** | *Enterprise IPv6 Deployment Guidelines*                 | Руководство по внедрению IPv6 в предприятиях | Полезно и для провайдеров, и для корпоративных сетей. |
| **BCP 214 / RFC 9099** | *Operational Security Considerations for IPv6 Networks* | Безопасность IPv6-сетей                      | Обязателен для проектирования безопасной сети.        |

## 3. Практические рекомендации от операторов (BCOP)

BCOP — это Best Current Operational Practice, документы от операторского сообщества, например RIPE.

| Документ     | Название (EN)                                                                                     | Перевод (RU)                                                                     | Применение                                                                                |
| ------------ | ------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| **BCOP 690** | *IPv6 prefix assignment for end-customers – persistent vs non-persistent and what size to choose* | Назначение IPv6-префикса клиентам — статическое или динамическое и выбор размера | Рекомендует выдавать /56 домашним клиентам, /48 бизнесу, использовать постоянный префикс. |
| **BCOP 674** | *IPv6 troubleshooting for helpdesks*                                                              | Диагностика IPv6 для техподдержки                                                | Практические советы для первой линии поддержки.                                           |
| **BCOP 641** | *IPv6 prefix assignment for small enterprises*                                                    | Назначение префиксов малому бизнесу                                              | Советы для обслуживания SMB-клиентов.                                                     |

## 4. Ключевые моменты при внедрении IPv6 у провайдера

1. **Адресное планирование**

   * Используйте **RFC 6177** (*IPv6 Address Assignment to End Sites*) — рекомендует не ограничивать клиентов префиксами длиннее /56.
   * Учитывайте рекомендации **BCOP 690** по выдаче статических префиксов.

2. **Развёртывание DHCPv6 и SLAAC**

   * Для массовых клиентов лучше SLAAC + DHCPv6-PD (**RFC 8415**) — это упростит конфигурацию.
   * Не используйте NAT66, если нет особой необходимости (**RFC 4864** — *Local Network Protection*).

3. **Поддержка переходных механизмов**

   * Для IPv4-зависимых сервисов — **NAT64/DNS64** (**RFC 6146**, **RFC 6147**).
   * Рассмотрите **464XLAT** (**RFC 6877**) для мобильных и IoT-сегментов.

4. **Безопасность IPv6**

   * Внедряйте фильтрацию по **BCP 38**, **BCP 84**.
   * Учитывайте особенности ICMPv6 (**RFC 4890**) — нельзя просто блокировать весь ICMPv6.

5. **Мониторинг и диагностика**

   * Используйте **RFC 5952** (стандартное представление IPv6-адресов) для логов.
   * Разверните тесты по **RFC 8212** (BGP-сессии только с фильтрацией).

## Заключение

Внедрение IPv6 у провайдера — это не только про настройку маршрутизаторов. Это комплексная задача, где важно учитывать рекомендации IETF, BCP и BCOP, планировать адресацию с запасом, обеспечить безопасность и удобство эксплуатации. Следуя приведённым документам, вы сможете построить надёжную и масштабируемую IPv6-сеть, готовую к будущему без IPv4.
